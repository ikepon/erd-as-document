# CLAUDE.md

このファイルはClaude Codeがプロジェクトの基本方針を理解するためのものです。

## 開発フロー

### ブランチ運用
- Issue ごとにブランチを作成
- 命名規則: `feature/issue-{番号}-{概要}`
  - 例: `feature/issue-1-setup`

### コミット
- Issue 内のチェックボックスごとにコミット
- コミット前にユーザーに確認を取る

### パッケージ管理
- パッケージのインストールは `npm install` を使用
- package.json を直接編集しない

## プロジェクト構成

```
erd-as-document/
├── package.json        # 統合（scripts + Nuxt）
├── scripts/
│   └── parse-schema.ts
├── app/                # Nuxt ソースディレクトリ
│   └── app.vue
├── public/             # 静的ファイル
└── nuxt.config.ts
```

- package.json: scripts と Nuxt の依存関係を統合
- app/: Nuxt のソースディレクトリ（コンポーネント、ページなど）

## コーディング規約

### TypeScript
- 型定義には `interface` ではなく `type` を使用する

## タスク実行ルール

### Issue のタスク管理
1. **Issue内の各タスク（チェックボックス）完了時のルール**:
   - 1つのチェックボックスが完了するごとにユーザーに完了を報告して確認を求める
   - ユーザー確認後、コミットを作成
   - 次のチェックボックスに進む

2. **実装の細分化ルール（重要）**:
   - **1つのコミットで実装する内容は最小限に留める**
   - 複数のタスクを一度に実装せず、1タスクずつ段階的に実装する
   - レビューしやすい粒度でコミットを分ける
   - 例: 1つのモジュールで複数機能を実装する場合も、機能ごとに分けてコミット

3. **コミットメッセージの形式**:
   - `[Issue #番号] タスク内容の簡潔な説明`
   - 例: `[Issue #15] Google Sheets API の設定と認証を実装`

4. **進捗報告の形式**:
   ```
   ✅ 完了したタスク: [タスク内容]
   📝 コミット: [コミットメッセージ]

   次のタスクに進んでよろしいですか？
   次のタスク: [次のタスク内容]
   ```
